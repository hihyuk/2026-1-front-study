3주차 스터디에서 JS와 HTML 방식의 한계를 극복하기 위해 React를 도입하는 배경을 공부했다. 기존 방식은 코드가 길어질수록 유지보수 비용이 증가하고 다수의 개발자가 협업하기 어려워지는 문제가 있었다. React는 화면의 UI를 컴포넌트 단위로 나누어 코드의 재사용성과 개발 효율성을 극대화하는 라이브러리이다. 

또, 자바스크립트 파일 내부에서 HTML을 직관적으로 작성할 수 있게 해주는 확장 문법인 JSX(Javascript Syntax eXtension)에 대해 학습했다. 이를 기반으로 페이지 전체를 새로고침하지 않고도 화면을 전환할 수 있는 SPA(Single Page Application)의 동작 원리도 알게 되었다.

실습 파트에서 터미널의 npx create-react-app 명령어를 이용해 프로젝트 초기 환경을 세팅하고 자기소개 페이지를 구성하는 여러 요소들을 개별 컴포넌트로 분리해 React의 구조적 편리함과 강력함을 체감했다. 프로젝트 실행 시 cd week3 로 이동한 후 npm start 명령어를 입력하면 localhost:3000가 실행된다.

JavaScript의 map 함수는 list의 모든 요소를 순회하며 주어진 규칙을 적용해 새로운 배열을 만들어 반환하는 메서드이다. React에서 이 함수를 이용해 동일한 형태의 컴포넌트나 태그를 화면에 반복해서 그릴 때 사용한다.
